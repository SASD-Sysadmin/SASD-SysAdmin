# SASD Sysadmin

Public Linux administration, automation and security resources by **SASD – Scientific and Software Development**.

This account collects practical material for system administration, Linux hardening, audit logging, automation and operational documentation.

## Focus Areas

- Linux server administration
- Debian and Ubuntu baseline configuration
- Security hardening
- auditd rules and file integrity monitoring
- Bash automation
- Ansible-based configuration management
- Monitoring and operational checklists
- Small-company sysadmin practices

## Planned Repositories

| Repository | Purpose |
|---|---|
| `linux-hardening-baseline` | Practical Linux hardening checklist and baseline configuration |
| `auditd-rules` | Comprehensive auditd rules for monitoring security-relevant system activity |
| `ansible-linux-baseline` | Ansible playbooks and roles for repeatable Linux configuration |
| `bash-admin-scripts` | Small, readable administration scripts for daily Linux work |
| `sysadmin-docs` | Runbooks, checklists and operational documentation |

## Principles

The material in this account follows a practical and conservative administration style:

- simple before clever
- readable before compact
- documented before magical
- security-conscious by default
- no secrets, customer data or environment-specific credentials
- suitable as learning material and as a foundation for small-business operations

## Legal and Security Notice

The content is provided for educational and operational preparation purposes.  
Before using scripts or configurations on production systems, review and test them carefully in a safe environment.

Never publish real credentials, private keys, customer names, internal IP addresses or sensitive infrastructure details.

## About SASD

SASD stands for **Scientific and Software Development**.  
The long-term goal is to build a practical, maintainable and well-documented toolbox for Linux administration, automation, monitoring and secure small-company IT operations.
